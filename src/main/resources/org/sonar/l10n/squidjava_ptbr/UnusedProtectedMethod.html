Los métodos protegidos que nunca se usan corresponden a código muerto. El código muerto es código inútil e inoperante que debe suprimirse. Esto simplifica el mantenimiento al disminuir el tamaño del código, y por ello simplifica el entendimiento del programa evitando la introducción de nuevos errores.

Dentro de los casos aplicables, Sonar no considera código muerto :

  • Métodos protegidos que redefinen un método de otra clase antecesora.
  • Métodos protegidos de una clase abstracta.
